## Milestone 2 (Priority 2) — Add SummarizedExperiment interoperability

**Branch:** `modernise/02-summarized-experiment`  (branch from `modernise/01-drop-plyr` after merge to `devel`)

### Background

`SummarizedExperiment` (SE) is the standard Bioconductor container for

Bioconductor workflow tool (DESeq2, edgeR, scater, mia, MicrobiomeStat)

### Changes required

#### New file: `R/SE-methods.R`

Create this file with the following content (full roxygen2 documentation required):

```r
#' Convert a biom object to a SummarizedExperiment
#'
#' Converts a \code{\link{biom-class}} object into a
#' \code{\link[SummarizedExperiment]{SummarizedExperiment}}, placing the
#' count matrix in \code{assay(., "counts")}, sample metadata in
#' \code{colData()}, and observation/feature metadata in \code{rowData()}.
#'
#' @param x A \code{\link{biom-class}} object.
#' @return A \code{SummarizedExperiment} object.
#' @seealso \code{\link{read_biom}}, \code{\link{biom_data}}
#' @export
#' @examples
#' biom_file <- system.file("extdata", "rich_sparse_otu_table.biom",
#'                          package = "biomformat")
#' x <- read_biom(biom_file)
#' if (requireNamespace("SummarizedExperiment", quietly = TRUE)) {
#'   se <- biom_to_SummarizedExperiment(x)
#'   se
#' }
biom_to_SummarizedExperiment <- function(x) {
  if (!requireNamespace("SummarizedExperiment", quietly = TRUE))
    stop("SummarizedExperiment is required. ",
         "Install with: BiocManager::install('SummarizedExperiment')")
  mat <- biom_data(x)
  smd <- sample_metadata(x)
  omd <- observation_metadata(x)
  # colData and rowData must be DataFrame; coerce safely
  col_data <- if (is.null(smd)) S4Vectors::DataFrame(row.names = colnames(mat))
              else S4Vectors::DataFrame(smd)
  row_data <- if (is.null(omd)) S4Vectors::DataFrame(row.names = rownames(mat))
              else S4Vectors::DataFrame(omd)
  SummarizedExperiment::SummarizedExperiment(
    assays  = list(counts = mat),
    colData = col_data,
    rowData = row_data
  )
}

# S4 coercion: as(x, "SummarizedExperiment")
# Registered only when SummarizedExperiment is available
if (requireNamespace("SummarizedExperiment", quietly = TRUE)) {
  setAs("biom", "SummarizedExperiment", function(from) {
    biom_to_SummarizedExperiment(from)
  })
}
```

#### New file: `R/tidy-methods.R`

```r
#' Coerce a biom object to a long-format tidy data frame
#'
#' Returns one row per (feature, sample) pair, with columns:
#' \code{feature_id}, \code{sample_id}, \code{count} (or \code{value}
#' for character data), plus any sample and observation metadata columns.
#' Column name conflicts between sample and observation metadata are
#' resolved by suffixing \code{_sample} and \code{_feature} respectively.
#'
#' @param x A \code{\link{biom-class}} object.
#' @param ... Ignored; present for S3 compatibility.
#' @return A \code{data.frame} in long (tidy) format.
#' @export
#' @examples
#' biom_file <- system.file("extdata", "rich_dense_otu_table.biom",
#'                          package = "biomformat")
#' x <- read_biom(biom_file)
#' df <- as.data.frame(x)
#' head(df)
as.data.frame.biom <- function(x, ...) {
  mat   <- as(biom_data(x), "matrix")
  smd   <- sample_metadata(x)
  omd   <- observation_metadata(x)
  # Convert to long form using base R (no tidyr dependency)
  long  <- as.data.frame(as.table(mat), stringsAsFactors = FALSE)
  colnames(long) <- c("feature_id", "sample_id", "count")
  # Left-join sample metadata
  if (!is.null(smd))
    long <- merge(long,
                  data.frame(sample_id = rownames(smd), smd,
                             check.names = FALSE, stringsAsFactors = FALSE),
                  by = "sample_id", all.x = TRUE, suffixes = c("", "_sample"))
  # Left-join observation metadata
  if (!is.null(omd))
    long <- merge(long,
                  data.frame(feature_id = rownames(omd), omd,
                             check.names = FALSE, stringsAsFactors = FALSE),
                  by = "feature_id", all.x = TRUE, suffixes = c("", "_feature"))
  long
}
```

## Milestone 4 (Priority 4) — Fix `generate_matrix()`: sparse-direct construction

**Branch:** `modernise/04-sparse-matrix-fix`  (branch from `devel` after M3 merge)
**Target version:** `1.39.8`

### Background

`generate_matrix()` in `R/BIOM-class.R` (lines ~774–792) is called on the
HDF5 read path. It currently:
1. Builds a full **dense** numeric matrix from the HDF5 CSR data
2. Row-by-row converts it to a list with `lapply(1:nrow(counts), function(i) counts[i,])`

This is deeply inefficient for large datasets (OTU tables with thousands of
features × hundreds of samples): it materialises the entire dense matrix in
memory before the sparse representation is even created.

### Exact replacement for `generate_matrix()`

```r
generate_matrix <- function(x) {
  # HDF5 BIOM v2 stores the count matrix as a CSR (compressed sparse row)
  # array: indptr (row boundaries), indices (column positions), data (values).
  # Construct a sparse Matrix directly from the CSR triplet without ever
  # materialising the full dense matrix in memory.
  indptr  <- as.integer(x$sample$matrix$indptr)   # length = n_samples + 1
  indices <- as.integer(x$sample$matrix$indices)  # 0-based column indices
  vals    <- x$sample$matrix$data                 # non-zero values

  n_obs     <- length(x$observation$ids)
  n_samples <- length(x$sample$ids)

  # Convert CSR indptr to per-element row index (0-based -> 1-based)
  # indptr[j+1] - indptr[j] = number of non-zeros in column j
  j_idx <- rep(seq_len(n_samples), diff(indptr)) # 1-based sample (col) index
  i_idx <- indices + 1L                          # 1-based observation (row) index

  m <- Matrix::sparseMatrix(
    i = i_idx, j = j_idx, x = as.numeric(vals),
    dims = c(n_obs, n_samples),
    dimnames = list(x$observation$ids, x$sample$ids)
  )
  # Return as a named list of row vectors for compatibility with
  # the rest of the biom object structure (biom_data() calls lapply on this)
  lapply(seq_len(nrow(m)), function(i) as.numeric(m[i, ]))
}
```

## Branch and merge protocol

For each milestone:

```bash
# Start from clean devel
git checkout devel
git pull upstream devel          # sync with Bioconductor before branching

# Create milestone branch
git checkout -b modernise/0N-short-name

# ... implement changes, tests, version bump, NEWS ...

# R CMD check must be clean before merge
R CMD build . && R CMD check --as-cran biomformat_*.tar.gz

# Merge back to devel (no-ff to preserve branch history)
git checkout devel
git merge --no-ff modernise/0N-short-name -m "Merge modernise/0N-short-name into devel"

# Push both the topic branch and updated devel to origin
git push origin modernise/0N-short-name
git push origin devel

# Push devel to Bioconductor (the BBS reads from here)
git push upstream devel
```

**Never force-push `devel` or `upstream/devel`.** All fixes go through topic
branches merged with `--no-ff`.

## Constraints and non-goals

- **Do not** change the `x.y` components of the version (`1.39`) — Bioconductor
  manages those at release cut time
- **Do not** change the S4 class definition in `allClasses.R` — S7 migration is
  a future major version
- **Do not** add `dplyr`, `tidyr`, `ggplot2`, or any other tidyverse package to
  `Imports` — they belong in `Suggests` at most
- **Do not** replace the BIOM file format itself with a database or Arrow backend
- After M5 (roxygen2 refresh), **never hand-edit NAMESPACE** — always go through
  `roxygen2::roxygenise()`
- All tests must use `testthat` 3.x idioms (`expect_true`, `expect_equal`, etc.)
  — `expect_that`/`is_true` are already removed and must not be reintroduced
- `R CMD check --as-cran` must finish with **0 ERRORs and 0 WARNINGs** before
  any merge to `devel`
